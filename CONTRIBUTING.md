# Contributing to Office Document Converter

Thank you for considering contributing to this project!

## Getting Started

1. **Fork** the repository on GitHub.
2. **Clone** your fork locally.
3. **Install dependencies**: `npm install`
4. **Create a branch** for your feature or bug fix: `git checkout -b feature/my-feature`

## Development Workflow

1. Make your changes in the `src/` directory.
2. If adding a new feature, add a test case in `tests/` (if available) or verify manually.
3. Run the CLI locally to test: `node bin/cli.js convert test-file.docx`
4. Ensure code style is consistent (standard JS style).

## Submitting Changes

1. **Push** your changes to your fork.
2. **Submit a Pull Request** to the main repository.
3. Provide a clear description of your changes.

## Reporting Issues

If you find a bug or have a feature request, please open an issue on GitHub.
Include:
- OS version
- Node.js version
- `unoconv` version (`unoconv --version`)
- Steps to reproduce the issue.

## Code of Conduct

Please be respectful and considerate of others when contributing.
